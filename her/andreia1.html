If you were to properly simulate these Pokémon in a software, you'd first need to define a Pokémon Class.

After that you'd create a type and name attributes to be chosen from a predefined set,

and then height and weight attributes that could vary randomly inside a given range for each Pokémon.

The nickmame attribute is often treated as optional, using the name of the Pokémon as a fallback nickname.

Once all that is taken care of all that would be left to do is define the methods to simulate the moves,

which could be simplified into being a single method that receives as arguments all the effects the move causes,

this way you'd save a lot of time, as you'd only need to specify the details of each attack,

instead of repeating the entirety of the logic for dealing dmg or getting a buff for example.
